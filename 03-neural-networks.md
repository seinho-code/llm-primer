# 03 神经网络模型

## 这章怎么读

如果你刚读完数学基础，这章可以看成“把前面的向量、矩阵、梯度真正装进模型里”。  
不要急着记各种网络名字，先抓住最核心的问题：模型为什么能从数据里学出表示。

读这章时，建议优先盯住 4 个动作：

- 输入怎么变成中间表示
- 中间表示怎么一层层变化
- loss 怎么定义“错了多少”
- 梯度怎么把错误反传回参数

## 先记住这条训练主线

```mermaid
flowchart LR
  A["输入 x"] --> B["线性层/激活"]
  B --> C["得到输出 y"]
  C --> D["计算 loss"]
  D --> E["反向传播"]
  E --> F["更新参数"]
```

神经网络是大模型的底座。要理解 Transformer 和 LLM，先要理解“为什么一个由矩阵乘法和非线性组成的大函数，能从数据里学出语言能力”。

## 1. 神经网络到底是什么

神经网络可以抽象成：

`y = f(x; theta)`

其中：

- `x` 是输入
- `y` 是输出
- `theta` 是所有可学习参数

模型训练的目标，就是找到一组参数，使模型在训练数据上的预测误差尽量小，并且在新数据上仍然有效。

## 2. 从感知机到多层网络

### 2.1 感知机

最简单的神经元可以看成：

`output = activation(w1*x1 + w2*x2 + ... + b)`

如果没有非线性激活，多层线性变换仍然等价于一层线性变换。因此，真正让网络表达复杂函数的是非线性层。

常见激活函数包括：

- `ReLU`
- `GELU`
- `SiLU`
- `tanh`

### 2.2 多层感知机

把很多层叠起来，就得到 MLP：

```mermaid
flowchart LR
  A["输入 x"] --> B["线性层"]
  B --> C["激活函数"]
  C --> D["线性层"]
  D --> E["激活函数"]
  E --> F["输出 y"]
```

深度带来的意义在于：网络可以分层抽象特征。前层学低级模式，后层学更复杂的组合。

## 3. 训练是怎么发生的

### 3.1 损失函数

模型输出后，需要一个目标衡量它“错得有多厉害”。这个目标就是 loss。

分类任务常见的是 `cross-entropy`，回归任务常见的是 `MSE`。

### 3.2 反向传播

反向传播利用链式法则，把输出端的误差逐层传回去，计算每个参数对 loss 的影响，也就是梯度。

### 3.3 梯度下降

拿到梯度后，用优化器更新参数。最常见的是：

- SGD
- Adam
- AdamW

更新直觉是：

`new_theta = old_theta - learning_rate * gradient`

## 4. 为什么神经网络比手工特征更强

手工特征方法依赖人告诉模型“什么重要”，而神经网络能自己学习中间表示。

例如情感分类里，传统方法可能手工设计：

- 是否出现“好评”
- 是否出现感叹号
- 否定词距离情感词多远

神经网络则直接从大量样本中学习这些模式，不必显式写规则。

## 5. Embedding：离散世界到连续世界的桥

大模型输入首先是 token ID。ID 本身没有语义，必须映射到向量空间。

这一步就是 Embedding。

假设词表大小是 `V`，隐层维度是 `d`，那么 embedding 矩阵大小约为：

`V x d`

每个 token 对应其中一行向量。训练后，语义相近的 token 往往在向量空间里更接近。

这一步非常关键，因为它把“离散符号处理”转换成“连续向量计算”。

## 6. 序列建模为什么难

文本不是独立样本，而是有顺序和上下文依赖的序列。

例如：

- “苹果很好吃”里的“苹果”偏水果
- “苹果发布新芯片”里的“苹果”偏公司

同一个 token 的含义取决于上下文。这迫使模型不仅学“词是什么”，还要学“词在句子中的角色”。

## 7. RNN/LSTM：Transformer 之前的主角

### 7.1 RNN

RNN 每一步读一个 token，并把隐藏状态传到下一步：

`h_t = f(x_t, h_(t-1))`

它的好处是顺序结构自然，问题是：

- 难并行
- 长距离依赖容易衰减
- 训练不稳定

### 7.2 LSTM / GRU

LSTM 和 GRU 通过门控机制缓解“记不住长信息”的问题。它们在 Transformer 之前非常重要。

但它们仍然保留了序列串行依赖，这在超大规模训练时是瓶颈。

## 8. 反向传播到底在传播什么

很多初学者会把反向传播理解成“一个神奇算法”。更准确地说，它只是链式法则在复合函数上的工程化实现。

假设一个两层网络：

`h = W1x`

`y = W2h`

`L = loss(y, target)`

那么我们真正需要的是：

- `dL/dW2`
- `dL/dW1`

因为只有拿到每个参数对损失的影响，优化器才能知道该往哪个方向更新。

反向传播做的事就是：

- 先算输出层误差
- 再把误差通过各层局部导数一路乘回去
- 在每一层累积梯度

从线性代数角度看，深度学习之所以适合 GPU，就是因为前向和反向都能被写成大量规则、重复、可批处理的矩阵运算。

## 9. 为什么 Embedding 是“查表”，却又能学出语义

Embedding 在实现层面常被写成一个查表操作，但从优化角度看，它仍是一个可学习参数矩阵。

假设词表大小为 `V`，隐层维度为 `d`，embedding 矩阵就是：

`E ∈ R^(V x d)`

输入 token ID `i` 时，本质上就是取矩阵第 `i` 行。训练过程中：

- token 出现在不同上下文里
- 该行向量不断被更新
- 最终让它在向量空间中占据“最适合帮助预测下一个 token”的位置

因此，embedding 不是在显式编码词典知识，而是在训练目标压力下形成的一种统计表示。

## 10. RNN/LSTM 为什么最终没有成为大模型底座

RNN 的问题不只在“记不住”，更在于它的计算图天然串行。

对长度为 `n` 的序列：

- RNN 必须从 `t=1` 算到 `t=n`
- 下一步依赖上一步隐藏状态
- 很难把整个序列在训练时完全并行化

LSTM 虽然通过输入门、遗忘门、输出门缓解了梯度消失，但没有改变这个串行本质。

这在小模型时代还能接受，在超大规模预训练时代就成了决定性瓶颈。

## 11. 深度学习为什么在 2010 年代爆发

不是因为理论突然改变，而是条件成熟了：

- GPU 让矩阵计算成本大幅下降
- 数据规模激增
- 训练技巧更成熟，例如初始化、归一化、优化器改进

语言模型的发展从某种意义上是“神经网络规模化”的结果。

## 12. 语言模型的核心训练目标

大多数自回归语言模型的目标都可以写成：

“给定前面的 token，预测下一个 token 的概率分布”

也就是：

`P(x_t | x_1, x_2, ..., x_(t-1))`

训练时通常最小化负对数似然：

`L = -Σ log P(x_t | x_<t)`

这个目标看起来简单，但它迫使模型同时处理：

- 局部语法一致性
- 跨句主题连续性
- 世界知识共现关系
- 程序语言中的精确符号模式

也正因为目标统一，模型才能把各种不同任务都投射到“续写”这个公共接口上。

## 13. 为什么“只做下一个 token 预测”会学出世界知识

这是 LLM 最反直觉、也最值得反复想的一点。

如果模型想把下一个 token 预测对，它就必须建模足够多的潜在变量，例如：

- 当前文本主题是什么
- 说话者意图是什么
- 哪些事实通常共同出现
- 哪种代码模式在这里最合理

这些变量不会在训练数据里显式标注出来，但会被内部表示隐式捕捉。你可以把它理解为：

模型不是直接被教“世界知识”，而是在压缩大规模文本分布时，被迫形成对世界结构的内部近似。

## 14. 神经网络阶段留下的关键遗产

在 Transformer 出现之前，神经网络已经确立了几件今天仍然有效的原则：

- 表示要从数据中学，不要完全靠手工设计
- 向量空间是语义运算的核心载体
- 端到端训练通常优于分阶段规则拼装
- 规模、数据和算力之间存在明显的耦合关系

## 15. 小结

神经网络带来的革命不是“换了个更复杂的分类器”，而是把 NLP 从离散规则世界推进到连续表示世界。Embedding 让离散符号进入向量空间，反向传播让多层表示能够被数据驱动地共同优化，RNN/LSTM 则提供了早期序列建模经验，而这些积累最终共同铺平了 Transformer 的道路。

## 16. 学以致用

如果你想把这一章真正吃透，可以做一个最小练习：

- 用纸笔画出 `token -> embedding -> hidden state -> logits` 的数据流
- 再解释清楚：反向传播时，误差是如何从 logits 一路传回 embedding 的

只要这两件事你能讲明白，后面读 Transformer 就不会觉得它是凭空冒出来的。
